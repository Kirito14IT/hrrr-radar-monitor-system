# Radar Care 设计规范

本文档记录当前项目采用的整体 UI 设计风格，供后续前端、Android App、演示页面和文档继续统一视觉语言时参考。

## 1. 设计定位

当前视觉方向定义为：

```text
Silver Shield / 浅色医疗看护风格
```

适用场景：

- 医院护士站多床位监护屏
- 养老院护理人员看护台
- 单床位实时生命体征与睡眠看护页面
- Android 移动端护理查看 App
- 比赛演示和项目展示页面

整体气质要求：

- 专业、干净、可信赖
- 医疗和护理场景友好
- 不使用强烈科技感深色大屏
- 不使用过多渐变、霓虹、发光和装饰
- 异常和紧急状态必须醒目，但正常状态应保持安静、克制

## 2. 视觉关键词

- 高明度冷灰背景
- 深蓝标题文字
- 品牌蓝作为唯一主强调色
- 蓝灰色辅助文字
- 白色软卡片
- 细边框
- 轻阴影
- 大留白
- 圆角但不过度可爱
- 红色只用于紧急/危险

## 3. 色彩规范

### 3.1 主色

| 用途 | 色值 | 说明 |
|---|---:|---|
| Brand Color | `#325BF2` | 主品牌蓝，用于主按钮、选中态、重点强调 |
| Primary | `#222E61` | 深蓝标题色，用于页面主标题、重要数字 |
| Secondary | `#7982A6` | 蓝灰辅助色，用于说明文字、次级信息 |

### 3.2 背景色

| 用途 | 色值 | 说明 |
|---|---:|---|
| 页面主背景 | `#F3F6FB` | Web 和 App 的基础背景 |
| 背景渐变浅色 | `#F8FAFF` | 页面上方和卡片外层的轻微提亮 |
| 背景渐变灰蓝 | `#EEF2F8` | 页面底部或大面积背景 |
| 卡片背景 | `#FFFFFF` | 所有业务模块卡片 |
| 卡片弱背景 | `#F8FAFF` | 卡片内部小块、输入框背景、统计块 |

### 3.3 边框与分割线

| 用途 | 色值 | 说明 |
|---|---:|---|
| 默认边框 | `#DFE5F0` | 卡片、输入框、表格边界 |
| 强边框 | `#C7D0DF` | 重要分区或激活边界 |
| 极弱分割线 | `#F0F3F8` | 表格行、列表项分隔 |

### 3.4 状态色

| 状态 | 色值 | 背景色 | 用途 |
|---|---:|---:|---|
| 正常 / 在线 | `#27B36A` | `#EAF8F1` | 在线设备、正常状态 |
| 关注 / 预警 | `#EE9B2D` | `#FFF7E8` | 需要关注但未到紧急 |
| 紧急 / 危险 | `#E23744` | `#FFF0F2` | 报警、摇晃、呼吸暂停、离床等 |
| 离线 / 禁用 | `#7982A6` | `#F2F5FB` | 离线设备、空状态 |

原则：

- 页面默认不出现大面积红色。
- 红色只用于真正需要护理人员立即处理的事件。
- 蓝色用于“可点击、当前选中、品牌重点”。
- 绿色只用于在线/安全/正常，不作为主品牌色滥用。

## 4. 字体规范

### 4.1 字体栈

Web：

```css
font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", "SF Pro Display",
  Roboto, "PingFang SC", "Microsoft YaHei", sans-serif;
```

Android：

```text
sans / Roboto 系统默认字体
```

### 4.2 字号层级

| 类型 | Web 建议 | App 建议 | 字重 |
|---|---:|---:|---|
| 大屏主标题 | 34–46px | - | 800–900 |
| 页面主标题 | 30–38px | 28–30sp | 800–900 |
| 模块标题 | 20–24px | 18–20sp | 700–800 |
| 指标大数字 | 26–36px | 22–24sp | 800–900 |
| 正文 | 15–16px | 15–16sp | 400–500 |
| 辅助说明 | 13–14px | 12–14sp | 400–600 |
| 标签 / Eyebrow | 12px | 12sp | 800–900 |

### 4.3 文字颜色

| 类型 | 色值 |
|---|---:|
| 标题 / 重要数字 | `#222E61` |
| 正文 | `#26324F` |
| 次级说明 | `#6C768F` |
| 弱提示 | `#7982A6` |
| 白色反显 | `#FFFFFF` |

排版原则：

- 标题使用深蓝，不使用纯黑。
- 正文避免过浅，保证医疗场景可读性。
- 标签类文本可使用大写英文或短中文，字距略大。
- 避免页面中同时出现过多字号，保持 4–5 个层级即可。

## 5. 间距与留白

### 5.1 页面级间距

| 场景 | 建议 |
|---|---|
| Web 内容区外边距 | 28–34px |
| 大屏页面外边距 | 34px |
| 移动端页面内边距 | 22–26dp |
| 页面模块间距 | 18–24px |
| 卡片网格间距 | 18–22px |

### 5.2 卡片内部间距

| 卡片类型 | 内边距 |
|---|---:|
| 大型 Hero 卡片 | 34–38px |
| 普通业务卡片 | 20–24px |
| 指标卡片 | 14–18px |
| 小标签 / Pill | 水平 10–14px，垂直 5–8px |

原则：

- 宁可多留白，不要把信息挤满。
- 护理人员需要快速扫视，模块之间要有明显呼吸感。
- 重要状态和指标要用间距分组，而不是靠大量颜色区分。

## 6. 组件规范

### 6.1 卡片

默认卡片：

```css
background: #ffffff;
border: 1px solid #dfe5f0;
border-radius: 24px;
box-shadow: 0 12px 32px rgba(34, 46, 97, 0.08);
```

卡片 Hover：

```css
transform: translateY(-4px);
box-shadow: 0 18px 42px rgba(34, 46, 97, 0.12);
```

注意：

- 不使用厚重黑色阴影。
- 不使用大面积毛玻璃。
- 卡片可有非常轻的浅蓝渐变，但不要显得花哨。

### 6.2 按钮

主按钮：

```css
background: linear-gradient(180deg, #4269ff, #325bf2);
color: #ffffff;
border-radius: 999px;
font-weight: 800;
box-shadow: 0 10px 20px rgba(50, 91, 242, 0.22);
```

次按钮：

```css
background: #ffffff;
color: #222e61;
border: 1px solid #dfe5f0;
border-radius: 999px;
```

危险按钮：

```css
background: #e23744;
color: #ffffff;
```

原则：

- 主按钮统一品牌蓝。
- 危险按钮只用于确认报警、解除紧急状态等高风险动作。
- 按钮圆角使用胶囊或大圆角，保持参考图的柔和组件感。

### 6.3 输入框

```css
background: #ffffff;
border: 1px solid #dfe5f0;
border-radius: 16px;
box-shadow: inset 0 0 0 1px #dfe5f0;
```

聚焦态：

```css
box-shadow:
  inset 0 0 0 1px rgba(50, 91, 242, 0.5),
  0 0 0 4px rgba(50, 91, 242, 0.10);
```

### 6.4 标签 / 状态 Pill

默认状态：

```css
background: rgba(50, 91, 242, 0.10);
color: #325bf2;
border-radius: 999px;
```

正常在线：

```css
background: #eaf8f1;
color: #126944;
```

预警：

```css
background: #fff7e8;
color: #9a5f11;
```

紧急：

```css
background: #e23744;
color: #ffffff;
```

离线：

```css
background: #f2f5fb;
color: #7982a6;
```

### 6.5 表格

表头：

```css
background: #f5f7fc;
color: #222e61;
font-weight: 800;
```

表格：

```css
border-radius: 18px;
border-color: #dfe5f0;
box-shadow: 0 6px 18px rgba(34, 46, 97, 0.06);
```

原则：

- 表格用于历史记录、事件列表、处理记录。
- 行分割线要浅。
- 不使用强烈斑马纹，最多使用极浅灰蓝背景。

## 7. 布局规范

### 7.1 Web 管理端

整体结构：

```text
左侧导航栏
  + 顶部状态栏
    + 页面内容区
```

左侧导航：

- 白色或近白背景
- 选中项使用品牌蓝实心背景
- 图标透明度较低，选中后反白
- 菜单项圆角 14px

顶部栏：

- 白色半透明
- 高度约 76px
- 放页面标题、当前床位/公共状态、报警入口

内容区：

- 浅灰蓝背景
- 页面左右留白 34px
- 卡片网格间距 18–22px

### 7.2 护士站页面

结构：

```text
Hero 统计区
  + 床位卡片网格
```

床位卡片应展示：

- 房间 / 床号
- 患者姓名
- 心率
- 呼吸率
- 温湿度
- 呼噜状态
- 设备在线状态
- 当前风险说明

视觉规则：

- 正常床位：白色卡片 + 蓝色状态
- 预警床位：浅黄色背景或边框
- 紧急床位：浅红背景 + 红色边框 + 轻脉冲
- 离线床位：灰蓝低饱和

### 7.3 监护大屏

结构：

```text
顶部标题 + 时间 + 声音授权按钮
统计条
重点关注条
床位卡片矩阵
```

设计要求：

- 大屏必须远距离可读。
- 标题和关键数字要更大。
- 床位图形可以保留，但颜色必须浅蓝、低噪声。
- 紧急事件床位必须标红。
- 紧急事件允许轻微闪烁或脉冲，但不能全屏炫光。

### 7.4 登录注册页

结构：

```text
左侧产品说明
右侧登录/注册卡片
```

视觉要求：

- 浅灰蓝背景
- 白色主卡片
- 品牌 Logo / Radar Care 标识
- 主标题深蓝、大字号
- 输入框和按钮与管理端保持一致

## 8. 动效规范

允许使用：

- 卡片 Hover 轻微上浮
- 正常床位轻微呼吸感
- 报警床位轻微红色脉冲
- 按钮 Hover 轻微上浮
- 页面元素淡入或轻微位移

避免使用：

- 大面积闪烁
- 强霓虹发光
- 快速循环动画
- 深色科幻扫描线
- 过多粒子、背景光斑

推荐动画参数：

```css
transition: transform 0.22s cubic-bezier(0.25, 0.46, 0.45, 0.94),
  box-shadow 0.22s ease;
```

紧急脉冲：

```css
box-shadow: 0 0 0 7px rgba(226, 55, 68, 0.10);
```

## 9. 告警规范

告警分级：

| 等级 | 颜色 | 展示方式 |
|---|---|---|
| 正常 | 绿色 / 蓝色 | 安静展示 |
| 关注 | 黄色 | 卡片边框或标签提示 |
| 紧急 | 红色 | 红色边框、红色标签、弹窗、大屏语音 |
| 离线 | 灰色 | 降低饱和度 |

紧急事件包括：

- 呼吸暂停
- 呼噜停止伴随呼吸异常
- 夜间疑似离床
- 求助语音
- 开发板摇晃
- 其它需要护理人员立即确认的事件

规则：

- 大屏报警必须标红对应床位。
- 语音播报内容应包含床位和事件信息。
- 告警红色区域要足够醒目，但不要让整个界面长期高频闪烁。
- 已处理事件不再显示为待处理报警。

## 10. Android App 规范

Android App 与 Web 保持同一视觉语言。

当前使用：

| 用途 | 色值 |
|---|---:|
| 主按钮 / Accent | `#325BF2` |
| 标题文字 | `#222E61` |
| 辅助文字 | `#7982A6` |
| 页面背景 | `#F3F6FB` |
| 卡片背景 | `#FFFFFF` |
| 卡片弱背景 | `#F8FAFF` |
| 边框 | `#DFE5F0` |
| 报警 | `#E23744` |
| 正常在线 | `#27B36A` |

组件要求：

- 页面背景为浅灰蓝。
- 顶部标题使用 `Radar Care`。
- 指标卡为白色圆角卡片。
- 按钮使用蓝色胶囊按钮。
- 输入框白底、细边框、大圆角。
- 系统状态栏和导航栏使用浅灰蓝。

## 11. 实现位置

Web 设计 token 和全局覆盖主要位于：

```text
frontend/src/assets/global.css
```

Android App 视觉样式主要位于：

```text
android_app/app/src/main/java/com/radarcare/guardian/MainActivity.kt
android_app/app/src/main/res/values/styles.xml
```

## 12. 后续维护建议

新增页面时应优先复用：

- `care-page-shell`
- `care-glass-card`
- 全局 CSS 变量
- Element Plus 默认组件
- 当前状态色规则

避免：

- 单页自行定义完全不同的主色
- 大面积深色背景
- 使用多个互相竞争的强调色
- 为装饰而添加复杂图形
- 在非报警场景使用红色

设计判断标准：

```text
如果护理人员在 3 秒内能看清：
1. 哪个床位异常
2. 异常是什么
3. 是否需要立即处理
4. 正常床位是否稳定

则该界面设计是合格的。
```

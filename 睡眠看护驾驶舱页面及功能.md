# 新增“睡眠看护驾驶舱”页面设计计划

## Summary
在现有“生命体征监测”和“历史数据”之外，新增一个独立前端页面：**睡眠看护驾驶舱**。页面面向夜间看护/展示场景，采用深色夜间霓虹大屏风格，基于雷达心率、呼吸率、距离、呼噜强度、设备在线状态和睡眠分期数据，新增 3 个核心功能：**睡眠质量评分、呼噜扰动地图、夜间守护事件流**。同时后端新增睡眠分析接口和 SQLite 事件表，支持“实时 + 历史”切换。

## 当前项目已有功能整理
- **模拟硬件联调**
  - `backend/mock_hardware_api.py`：FastAPI 模拟后端，提供登录、注册、实时数据、历史数据、AI 分析、音频接收、timeline 等接口。
  - `backend/mock_device_sender.py`：模拟毫米波雷达板和呼噜检测板持续发送数据。
- **实时生命体征页**
  - `frontend/src/views/heart_pic.vue`
  - 展示雷达/呼噜板在线状态、心率、呼吸率、呼噜强度三图对齐、声浪条、睡眠分期、呼噜-生命体征关联、实时健康摘要。
- **历史数据页**
  - `frontend/src/views/data.vue`
  - 展示历史心率/呼吸率表格，调用后端 `/ai/analyze-vitals` 做 DeepSeek 或本地规则分析。
- **当前可用数据**
  - `/timeline`：秒级心率、呼吸率、距离、呼噜分数、音量、呼噜强度、睡眠分期、设备在线状态。
  - `/status`：当前设备状态、呼噜事件、音频上传次数。
  - `/heartdata/selectPage`：历史心率/呼吸率分页数据。

## 新页面核心功能

### 1. 睡眠质量评分
新增大屏首页核心指标：`Sleep Quality Score`，0-100 分。
- 数据来源：最近 30 分钟实时 timeline 或历史事件表。
- 评分因素：
  - 心率是否稳定。
  - 呼吸率是否稳定。
  - 呼噜强度和呼噜次数。
  - 是否无人/离床。
  - 雷达板或呼噜板是否掉线。
- 可视化：
  - 中央发光圆环仪表盘。
  - 分数旁边显示状态：`稳定睡眠`、`轻度扰动`、`呼噜频繁`、`疑似离床`、`设备异常`。
  - 下方显示扣分原因，例如“呼噜扰动 -12，呼吸波动 -8，设备离线 -5”。

### 2. 呼噜扰动地图
新增“时间热力条”，展示某段时间内呼噜对睡眠的影响。
- 数据来源：`snore_level`、`snore_detected`、`heart_rate`、`breath_rate`。
- 计算逻辑：
  - 每分钟聚合一次平均呼噜强度。
  - 标记呼噜发生点。
  - 对比呼噜前后心率/呼吸变化，生成扰动等级。
- 可视化：
  - 横向时间轴热力图。
  - 低扰动为蓝色，中度为紫色，高扰动为橙红色。
  - 鼠标悬浮显示：时间、呼噜分数、音量 dBFS、心率变化、呼吸变化。
  - 页面顶部显示“最严重扰动时刻”。

### 3. 夜间守护事件流
新增类似“监控中心日志”的事件面板。
- 数据来源：设备在线状态、`target_distance`、`sleep_stage`、呼噜事件、异常生命体征。
- 事件类型：
  - `呼噜事件`：呼噜强度超过阈值。
  - `疑似离床`：雷达无人或距离异常。
  - `呼吸异常`：呼吸率过高/过低或剧烈波动。
  - `心率异常`：心率过高/过低或剧烈波动。
  - `设备离线`：雷达板/呼噜板停止发送。
  - `恢复正常`：设备或体征恢复。
- 可视化：
  - 右侧竖向时间线。
  - 每类事件使用不同霓虹颜色和图标。
  - 支持“全部 / 只看异常 / 只看呼噜 / 只看设备”过滤。

## Key Changes

### 前端
- 新增页面：`frontend/src/views/sleep_dashboard.vue`
- 新增路由：`/manage/sleep_dashboard`
- 左侧菜单新增：`睡眠看护驾驶舱`
- 页面布局：
  - 顶部：实时/历史模式切换、时间范围选择、设备状态胶囊。
  - 中央：睡眠质量评分圆环 + 核心摘要。
  - 左侧：呼噜扰动地图。
  - 右侧：夜间守护事件流。
  - 底部：心率/呼吸/呼噜稳定性小卡片。

### 后端
- 新增接口：
  - `GET /sleep/overview?mode=live&seconds=1800`
  - `GET /sleep/overview?mode=history&date=YYYY-MM-DD&userID=1`
- 返回内容包括：
  - 睡眠质量评分。
  - 扣分原因。
  - 每分钟聚合的呼噜扰动热力数据。
  - 夜间事件流。
  - 心率/呼吸/呼噜稳定性统计。
- 新增 SQLite 表：
  - `sleep_events`
  - 保存呼噜事件、离床事件、设备离线事件、异常体征事件。
- 后端继续兼容现有 `/timeline`、`/status`、`/heartdata/selectPage`，不破坏原页面。

## 可选设计方向
推荐采用方案 A。

- **方案 A：夜间看护驾驶舱（推荐）**
  - 睡眠质量评分 + 呼噜扰动地图 + 夜间事件流。
  - 最贴合“应用场景”和当前数据。
  - 适合答辩、展示、师兄接手扩展。
- **方案 B：开发板联调中心**
  - 展示数据包、延迟、丢包、接口调用、设备心跳。
  - 更适合硬件开发，但不如看护页面炫酷。
- **方案 C：健康风险雷达**
  - 做风险评分、风险雷达图、AI 报告。
  - 展示感强，但和当前实时模拟数据的关系弱一些。

## Test Plan
- 后端：
  - `python -m py_compile backend\mock_hardware_api.py backend\mock_device_sender.py`
  - 启动后端后访问：
    - `/sleep/overview?mode=live&seconds=1800`
    - `/timeline?seconds=1800`
    - `/status`
  - 启动雷达板和呼噜板，确认 sleep overview 分数、热力图、事件流随数据变化。
  - 关闭呼噜板，确认事件流出现呼噜板离线，心率/呼吸仍继续。
  - 关闭雷达板，确认出现雷达离线或疑似离床事件。
- 前端：
  - `cd frontend`
  - `npm run build`
  - 打开 `/manage/sleep_dashboard`
  - 验证实时/历史切换、评分圆环、热力图、事件流、过滤器正常。
- 数据持久化：
  - 后端重启后，历史模式仍能读取之前保存的睡眠事件。
  - `.env`、数据库、音频、构建产物不提交。

## Assumptions
- 新页面命名为“睡眠看护驾驶舱”。
- 视觉采用深色夜间霓虹大屏。
- 新增 3 个核心功能：睡眠质量评分、呼噜扰动地图、夜间守护事件流。
- 支持实时 + 历史切换。
- 历史数据通过新增 SQLite 睡眠事件表保存。
- 不改真实硬件算法，只增强模拟模式、后端分析接口和前端展示。

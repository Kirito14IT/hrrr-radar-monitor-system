# 历史数据页重设计 + AI 分析持久化 + 新增“看护预警中心”页面

## Summary
- 根因：`D:\Docs\Embedded\HRRR-Radar\radar-monitor-system\frontend\src\views\data.vue` 把 `AI分析` 的 loading/report 状态放在页面局部 `reactive` 中；跳转离开后组件卸载，异步结果写回旧实例，新进入页面自然看不到过程或结果。
- 设计结论：保留已有“睡眠看护驾驶舱”，新增第 4 个侧栏页面：**看护预警中心**，面向夜间照护/异常响应场景。
- 当前 `npm run build` 与 `python -m py_compile` 已通过；Chrome 插件当前无法通信，后续执行完成后优先用浏览器手动/自动回归验证。

## Key Changes
- **AI 分析状态修复**
  - 新增 Pinia store：`D:\Docs\Embedded\HRRR-Radar\radar-monitor-system\frontend\src\stores\historyAnalysisStore.js`。
  - 将历史表格查询条件、表格数据、分页、AI 报告、provider、fallback、loading、error、lastAnalyzedAt 移入 store。
  - 用模块级 `inFlightAnalysisPromise` + `requestToken` 保持跨路由异步过程：离开 `/manage/data` 后请求继续；返回页面仍显示“分析中”或最终报告；重置/新请求会忽略旧结果。

- **重设计 `/manage/data`**
  - 重写 `D:\Docs\Embedded\HRRR-Radar\radar-monitor-system\frontend\src\views\data.vue` 为“历史数据与 AI 洞察”布局。
  - 增加顶部说明区、数据质量 KPI、平均心率/呼吸率、异常计数、最近分析状态。
  - 表格保留原字段，但对心率/呼吸率增加正常/偏高/偏低标签；无数据时给启动模拟板与积累历史数据的引导。
  - AI 区改为独立洞察卡：loading skeleton、provider badge、fallback badge、错误重试、报告正文。

- **新增“看护预警中心”页面**
  - 新增视图：`D:\Docs\Embedded\HRRR-Radar\radar-monitor-system\frontend\src\views\alert_center.vue`。
  - 新增路由：`/manage/alert_center`，更新 `D:\Docs\Embedded\HRRR-Radar\radar-monitor-system\frontend\src\router\index.js`。
  - 更新侧栏：`D:\Docs\Embedded\HRRR-Radar\radar-monitor-system\frontend\src\views\manage.vue`，新增菜单项“看护预警中心”。
  - 页面功能：
    - 风险总览：基于 `/sleep/overview`、`/timeline`、`/status` 计算“立即关注 / 继续观察 / 稳定”。
    - 异常归因矩阵：心率、呼吸、呼噜、离床/无人、设备离线 5 类风险。
    - 照护动作队列：生成可执行建议，如“检查雷达距离”“确认呼噜板在线”“观察呼吸波动”。
    - 阈值策略面板：心率高低阈值、呼吸高低阈值、呼噜阈值、设备离线秒数，保存在 Pinia persisted state。
    - 趋势小窗：用 ECharts 展示最近 30 分钟心率/呼吸/呼噜小趋势。

- **统一 UI 风格**
  - 在 `D:\Docs\Embedded\HRRR-Radar\radar-monitor-system\frontend\src\assets\global.css` 增加语义 token：背景、surface、primary、success、warning、danger、radius、shadow。
  - 采用统一“医疗科技 + 玻璃卡片 + 深蓝绿高对比”风格，旧历史页和新预警页保持同一视觉语言。
  - 遵守 `ui-ux-pro-max` 要点：44px 以上触控目标、可见 focus、颜色不单独表达风险、空状态/加载态明确、图表有文字摘要。
